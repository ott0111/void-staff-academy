Ticket Handling Standards

Your priority is efficient, professional ticket management. Every response must follow protocol: greeting → purpose → age verification → correct routing → documentation.

Protocol Snapshot

Ticket Types General • Roster
Required Opening Professional greeting + role inquiry + age verification
Escalation Complex or sensitive cases go to senior staff with proof + context

Ticket Types

Correct classification = faster handling + cleaner logs

General Ticket

Created for support inquiries, moderator assistance, or reporting issues. Handle professionally and escalate complex matters with documentation.

Roster Ticket

Application channel for users seeking to join Void Esports. First response must include age verification and direction to proper resources.

Protocol (Non-Negotiable)

  • Start with a professional greeting (no “yo”, “bro”, etc.).
  • Ask what they need / what role they’re applying for.
  • Ask for age verification clearly.
  • Request correct proof (tracker/clips/socials/portfolio).
  • Route to correct department + keep records.

Approved Openers

Professional tone only

Greetings

  • Hello, how may I help you?
  • Hello, what can I assist you with today?

Core Questions

  • What role are you applying for today?
  • May I ask how old you are?

Role-Specific Routing

Ask for proof + ping the right department

Creative

  • @Creative Academy / @Main Creative — Ask for 2–3 clips and ping @Creative Department.

Roster

  • @Academy Roster — Ask for tracker + default dance lobby image.
  • @Semi-Pro / @Pro Roster — Ask for tracker + default dance lobby image and ping @operations.

GFX / VFX

  • @VOID VFX / @VOID GFX — Ask for portfolio/previous work and ping @GFX / VFX Lead.

Content

  • @Content Creator — Ask for socials and ping @Content Department.
  • @Coach — Ask who they coached + link trackers.

Fallback Offer: Grinder+

If they don’t qualify for roster roles, you can offer Void Grinder+. (Full requirements are listed in the Roster module.)

Closing the Ticket

Clean close + confirmation + roles

Final Step Prompt

“Perfect! Please change your name to have Void in it and send a screenshot of using our code!”

After Proof is Sent

“Thank you! Here are your roles! Have a nice day!”
